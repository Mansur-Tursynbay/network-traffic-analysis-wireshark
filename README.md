# Network Traffic Analysis using Wireshark

## Overview
This project focuses on analyzing network traffic using Wireshark to understand how devices communicate within a network.

## Objectives
- Analyze ARP, ICMP, and Ethernet traffic
- Understand MAC and IP address relationships
- Explore how packets move through the network

## Tools Used
- Wireshark

## Practical Work
- Captured network traffic in a controlled lab environment
- Analyzed ARP requests and responses
- Investigated ICMP packets (ping communication)
- Applied filters to isolate specific traffic
- Examined source and destination MAC/IP addresses

## Key Findings
- ARP is used to resolve IP addresses to MAC addresses
- ICMP is essential for connectivity testing
- Network traffic can be filtered and analyzed for troubleshooting and security analysis

## Skills Gained
- Network traffic analysis
- Packet inspection
- Understanding of TCP/IP communication
- Basic network troubleshooting

## Conclusion
This project improved my understanding of real network communication and strengthened my ability to analyze traffic for cybersecurity tasks.
